피드백 반영

저를 소개합니다

사용자 경험을 고려하되, 데이터 구조와 흐름을 함께 고민하며 성장해왔습니다.

학업과 다양한 프로젝트를 통해 기능 구현에 그치지 않고 문제를 정의하고 구조적으로 해결하는 과정을 중요하게 생각하게 되었습니다.

웹·모바일 서비스를 개발하며 단순한 화면 구현을 넘어 데이터 흐름과 기능 간 연계 구조를 함께 고민하는 개발 방식을 익혔습니다.
서비스의 동작 원리를 이해하기 위해 데이터가 어떻게 저장되고 연결되는지에 대해 꾸준히 탐구해왔습니다.

개발에서 중요하게 생각하는 가치는 정확한 설계, 문서화, 협업입니다.
설계 의도를 정리하고 공유하는 과정을 통해 팀 프로젝트의 완성도를 높이는 경험을 쌓았습니다.

센서 기반 인터랙션, 데이터 분석, 웹·모바일 프로젝트 등 다양한 경험을 통해 기술 선택의 기준과 구조적인 해결 방법을 고민해왔습니다.

데이터 구조를 정리하고 기능 간 흐름을 문서화하며 무결성과 일관성을 유지하는 설계의 중요성을 배우게 되었습니다.

단순히 동작하는 코드를 작성하는 것을 넘어 서비스의 기반을 이해하고 설계할 수 있는 개발자로 성장하고 있습니다.

StraySpotter

React Native JavaScript MySQL Firebase Auth Naver Map API Figma

핵심 역량 & 사례

저의 핵심 역량을 실제 사례로 정리했습니다.

💡 문제 해결

  • 학사 출석 관리 시스템 오류 해결: 출석 체크 시 중복 저장 문제를 발견하여 데이터 검증 로직 추가 → 중복 오류를 제거.
  • 노인 치매 예방 게임 정확도 향상: Skeleton 인식이 불안정했던 구간을 인식 안정화를 위한 처리 로직 추가.
  • 여행 데이터 분석 자동화: Google Trends / DataLab 데이터를 자동 병합하는 파이프라인 구성 → 분석 시간 50%↓.

🤝 협업 중심

  • WheelGo 팀 프로젝트: 기획·개발을 담당하며 팀원(백엔드·개발·디자인) 간 요구사항 조율 및 화면 흐름 정리.
  • 유기견 StraySpotter 팀 경험: 지도 기반 기능·데이터 흐름을 문서화해 개발자 간 이해 차이 제거.
  • 회의·결정 히스토리 정리: 팀플 진행 시 Notion으로 회의 내용/업무 분배를 기록해 협업 투명성 강화.

📚 꾸준한 학습

  • 웹 개발 학습 과정: HTML/CSS/JS → JSP·MySQL → React 중심의 프론트엔드 개발 경험을 확장.
  • 데이터 분석 스킬 향상: Pandas 기반 데이터 분석 / 시계열 분석 학습 후 여행트렌드 분석 프로젝트에 즉시 적용.
  • Kinect·Skeleton 처리 학습: 생소한 기술이었지만 문서·샘플 코드 분석을 통해 게임 완성까지 독자 구현.

💾 데이터 설계 & 구조 이해

  • 데이터 구조 설계 경험 : 모바일 및 웹 프로젝트에서 데이터 흐름을 정의하고, 중복을 최소화하도록 구조를 설계.
  • 데이터 일관성 확보 : 출석 관리 기능 구현 과정에서 중복 저장 문제를 분석하고 입력 단계의 검증 로직을 추가해 데이터 일관성을 유지.
  • MySQL · Firebase 활용 : MySQL 기반 프로젝트에서 관계형 데이터 흐름을 경험하고, Firebase Realtime Database를 활용해 데이터 동기화 구조를 설계.
